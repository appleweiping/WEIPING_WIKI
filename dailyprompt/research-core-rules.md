# 科研项目核心规则（永久执行）

> 适用范围：所有科研项目，长期有效，所有 agent 必须遵守。

## 核心规则

- **禁止 toy 化**：Opus 智商高但不能冲动，遇到复杂问题先想好思路再开始工作。GPT-5.5（Codex）作为 reviewer。
- **Baseline**：官方版本共 8 个，必须使用 official 顶会论文代码。有自己的创新非缝合方法。
- **严格按照开发规则和 ARIS 流程执行**。
- **每次改动**：自行从本地 commit + push 到 GitHub，同时更新 agentmemory 和该项目的重要文档，让所有 agent 都知道。
- **隐私保护**：注意避免隐私内容（API key、学号等）泄露到 GitHub。
- **Todo 列表**：维护完整 todo 列表。当用户在运行时继续添加 prompt，不得遗忘原有 todolist，而是在原有基础上增加新任务。
- **异常汇报**：出现 API key 不可用、服务器空间不足等问题要及时汇报。
- **实验监控**：每两小时监控一次实验进度，直到写作完成。
- **断线续接**：电脑即使关机重连，后续 agent 或聊天也能持续知道该如何接手实验（依赖 agentmemory + 项目文档）。
