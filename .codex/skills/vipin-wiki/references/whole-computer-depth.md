# Whole-Computer Depth Policy

Use this when a task touches local project routing, file organization, or whole-computer maintenance.

## Default Rule

Start shallow across `C:/`, `D:/`, and `G:/`. Promote a root to deeper inspection only when live evidence shows active use, high consequence, repeated user interest, or operating-system/agent infrastructure relevance.

Physical file organization is delegated to `D:\agent-resources\skills\vipin\workstation-maintenance`. Use that skill for dry-run manifests, age-gated move plans, approved batches, and rollback; use `vipin-wiki` for public-safe routing pages after dry-run evidence or actual state changes.

## Evidence To Read

For important roots, read only entry evidence first:

- `README*`, `AGENTS.md`, `CLAUDE.md`, `OPENCODE.md`
- `package.json`, `pyproject.toml`, solution/project manifests
- top-level `docs/`, `src/`, `apps/`, `packages/` names
- `git status --short --branch`
- existing wiki pages and catalog search results

Do not deep-read source trees, private files, datasets, logs, browser profiles, DBs, or generated artifacts during routing maintenance.

## Tier Handling

- Tier 0 operating roots get current commands, safety boundaries, validation gates, and links.
- Tier 1 active projects get purpose, path, status, first-read files, related wiki pages, and safety notes.
- Tier 2 archives get content nature, discovery clues, and edit caution.
- Tier 3 bulk/system roots get a bucket line and a reason for low detail.
- Private/sensitive roots get private-only minimal metadata or no public record.

No tier allows broad physical moves under `D:\Research`; resolved paths under that root are excluded from workstation move plans.

## Promotion Triggers

Promote a root when:

- the user asks about it repeatedly
- it has current README/AGENTS/package evidence
- it appears in recent logs, agentmemory, or active git work
- it affects agent infrastructure, research routing, healthcare/legal/account safety, or public publishing

Demote or archive a root when live evidence shows it is stale, duplicated, scratch-only, or no longer useful for routing.
